The Problem It Creates

"When there's no structure, time fills up with what's urgent — not what's impactful."
  • Analyst time has been going predominantly to execution tasks
  • Infrastructure work keeps getting deprioritised as a result
  • Marketing doesn't yet have the full picture it needs to make decisions

This is a structural issue — not a people one.

How Requests Work

The test: Does this require cross-system thinking, data modelling, or technical expertise the marketing team doesn't have? If yes → Tier 1.
1
Ops Analysts
Complex builds, infrastructure, analysis.
Example: attribution model build.
2
Ops Associates
Documented, repeatable tasks.
Example: post-event data upload.
3
Self-Serve
Documented processes the team runs.
Example: pre-event form testing.

All via the ops request form — routed internally.

Tier 3 — Self-Serve

A lot of this is already happening — we're just formalising it.

  • Pre-event form testing — checklist in the events Notion page, run before every event
  • On-the-day support — break/fix runbook in Notion, escalate to ops only if unresolved
  • Sales rep updates on the event demo form — Loom recorded, under 2 minutes
  • Leaderboard updates — built and documented, transitioning to events/community team
  • Standard campaign list pulls — saved views configured in HubSpot
  • Paid performance reporting — dashboards built, read your own channel data
  • Basic HubSpot reporting — open the dashboard before raising a request
